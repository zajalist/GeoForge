# GPlatesPython Simulation Skeleton

This folder contains non-implemented skeletons for migrating the app to `gplatespython`.

## Intended workflow

1. Use the exported `.gpml` generated by the UI as the initial feature input.
2. Implement loading and validation of GPML features.
3. Assign or resolve plate IDs and topological relationships.
4. Build reconstruction trees / rotations.
5. Run forward time steps using gplatespython primitives.
6. Export updated GPML and reconstruction outputs.

## Files

- `pipeline_skeleton.py`: High-level orchestration entry points.
- `gpml_io_skeleton.py`: GPML loading/saving placeholders.
- `reconstruction_skeleton.py`: Plate motion/reconstruction placeholders.

All callable methods intentionally raise `NotImplementedError`.
